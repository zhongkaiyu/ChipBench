I would like you to implement a module named TopModule with the following
interface. All input and output ports are one bit unless otherwise
specified.

 - input  clk
 - input  rst_n
 - input  data_in (8 bits)
 - input  valid_a
 - input  ready_b
 - output ready_a
 - output valid_b
 - output data_out (10 bits)

The module should implement a serial data accumulation circuit that receives
8-bit input data and outputs the accumulated sum of every 4 input values.
The circuit uses valid-ready handshake mechanisms for both input and output
interfaces to ensure efficient data transfer without performance loss.

The data accumulation should work as follows:
- Input interface: Uses valid-ready handshake mechanism
  - valid_a indicates the validity of input data_in
  - ready_a indicates the module is ready to receive upstream data
- Output interface: Uses valid-ready handshake mechanism
  - valid_b indicates the validity of output data_out
  - ready_b indicates downstream is ready to receive data

Data accumulation process:
- The module receives 8-bit input data_in when valid_a is high and ready_a is high
- After receiving 4 input data values, the module outputs the accumulated sum
  as 10-bit data_out with valid_b asserted
- The output data_out represents the sum of the 4 most recently received inputs
- Both input and output interfaces must support backpressure through ready signals
- The design must ensure no performance loss when both upstream and downstream
  are ready for full-speed transmission (no bubbles in data flow)
