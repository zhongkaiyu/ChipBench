I would like you to implement a module named TopModule with the following
interface. All input and output ports are one bit unless otherwise
specified.

 - input  clk
 - input  rst_n
 - input  read_en
 - input  write_en
 - input  read_addr (8 bits)
 - input  write_addr (8 bits)
 - input  write_data (4 bits)
 - output read_data (4 bits)

Implement a dual-port RAM with depth of 8 and data width of 4 bits. All
data should be initialized to 0000. The RAM has two separate ports: one
for reading data and one for writing data. Read and write operations can
be performed simultaneously.

When the read enable signal read_en is asserted, the module should read
data from the location specified by read_addr and output it on read_data.
When the write enable signal write_en is asserted, the module should write
write_data to the location specified by write_addr.

The rst_n signal is an active-low asynchronous reset signal. Assume all
sequential logic is triggered on the positive edge of the clock.


Based on the problem description above, the code below in TopModule has bug, please fix that:
```
module TopModule(
	input clk,
	input rst_n,
	
	input write_en,
	input [7:0]write_addr,
    input [3:0]write_data,
	
	input read_en,
	input [7:0]read_addr,
	output reg [3:0]read_data
);
    reg [3:0] myRAM [7:0];
    
    reg [8:0] i;
    always@(posedge clk or negedge rst_n) begin
        if(~rst_n)
            for(i=0;i<256;i=i+1)
                myRAM[i] <= 0;
        else
            myRAM[write_addr] <= !write_en? write_data: myRAM[write_addr];
    end
    always@(posedge clk or negedge rst_n) begin
        if(~rst_n)
            read_data <= 0;
        else
            read_data <= read_en? myRAM[read_addr]: read_data;
    end
endmodule
```
